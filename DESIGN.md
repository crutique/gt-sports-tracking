# Baseball Tracking GT — Art Direction

*Display name is **Baseball Tracking GT** (owner rename 2026-07-23, from "GT Summer Tracker"; nav brand splits it "BASEBALL TRACKING · GT" with GT in gold). The repo/deploy slug `gt-sports-tracking` is unchanged. The footer's "unofficial fan project" blurb was removed the same day; only the "Data last updated" stamp remains.*

*Direction A, "Under the Lights," approved 2026-07-21 after specimen review (A vs. dossier vs. scoreboard). Tuning from contrast pairs: navy carries heroes and signature displays only; the field-as-canvas motif recurs across surfaces; no grafts from the losing directions. This document is the authored source of truth — extraction tools (`/impeccable init`) describe what IS and are corrected against this, never the reverse.*

## Direction statement

Savant credibility wearing GT colors under stadium light. Navy heroes and illustrative, field-shaped data displays make the site unmistakably Georgia Tech's; dense white reference tables keep it unmistakably trustworthy. Numbers arrive like a broadcast score-bug — instantly, then get out of the way.

## References & extracted principles

- **baseballsavant.mlb.com** — one encoding system (blue-cold → red-hot) carried identically on every surface; percentile position as the universal comparison; credibility through density + restraint. *Extracted: the ramp is a language, not a decoration.*
- **mlb.com/gameday** — the field itself as the data canvas; diagrammatic baseball. *Extracted: geometry of the game does IA work (approved as a recurring motif).*
- **ramblinwreck.com** — GT's own voice: heavy condensed all-caps navy display, gold as band/accent, white field. *Extracted: the type register that reads "genuinely GT" (we fix their gold-on-gold contrast sins).*
- **ESPN (score bugs / broadcast graphics)** — state changes announced in ~150ms cues; information appears already-composed, never assembling itself. *Extracted: motion as state, zero choreography.*
- **NYT The Upshot (sports graphics)** — the annotation layer: graphics that say what matters (leaders, change, context) instead of only showing data. *Extracted: every signature display states its takeaway.*

## What we transpose

**Keep (quality DNA):** Savant's ramp-as-language and percentile idiom; Gameday's field-as-canvas; GT's condensed-caps navy/gold register; broadcast-fast state motion.
**Deliberately change:** typeface family (Barlow Condensed, not GT's licensed faces or Savant's system voice); the ramp's anchor colors (ours are AA-safe under white text, enforced by test); the field rendered as a flat illustration (muted grass/dirt tones, clean white lines — MLB-app style; no textures, no photorealism) used only where position is the encoding; gold restricted to identity + emphasis (never a text-bearing background at body sizes).

## Typography system

- **Display: Barlow Condensed** (500/600/700, @fontsource, OFL). All-caps for page headlines, hero names, section energy; sentence case allowed in annotations. Letter-spacing +0.01em at display sizes; never below −0.02em. Scoreboard numerals (stat cells, trend values) use 600.
- **Body/UI: the native system stack** — argued for, per product register: 12.5–15px data-dense tables render best on native rendering, cost zero bytes, and keep the display layer's contrast high. Identity lives in the display layer, not the body.
- **No third family.** (The dossier's mono-citation voice was considered and declined — receipts stay in the body face, muted.)
- Scale: 34–36px h1 · 26px scoreboard numerals · 15/14px body · 12.5px table · 11px floor for labels (10px only for uppercase table headers). `tabular-nums` on all stat cells.

## Color system

Tokens live in `site/src/styles/global.css`; the ramp's anchors mirror `site/src/lib/colors.ts` and are contrast-tested in `tests/colors.test.ts` (≥4.5:1 under white text at every percentile — a hard invariant).

- **Dominant: GT navy** `#003057` (+ `--navy-dark #00223f`, `--navy-bright #00406f`) — carries heroes and signature displays only; body surfaces are `--bg #f6f7f9` / `--card #fff`.
- **Identity accent: Tech gold** `#b39051` (re-anchored 2026-07-23 per owner; + light `#d9b677`, deep `#8f6f38`, derived with the original family's lightness offsets — all navy-on-gold / gold-on-navy pairs re-verified ≥4.5:1) — borders, card frames, brand marks, section rules. Never body-text-on-gold.
- **Data ramp:** `#3566cc` (cold) → `#5f6d83` → `#c53022` (hot); red = good, Savant convention.
- **Categorical league palette** (`--lg-*`, added 2026-07-23 per owner for the By league map): seven muted heritage tones — navy (CCBL), bronze (NWL), pine (APPY), steel (NECBL), teal (SFCBL), wine (CPL), rust (SBL). CATEGORICAL ONLY, never the performance ramp; each is dark enough for white text and reads on the map land. The one place categorical color is sanctioned — every other surface's color is navy/gold identity or the ramp.
- **Status:** ok `#e2efe4/#1e5b2a` · warn `#fdf3dd/#92600a` · bad `#f3e9e9/#7a3535` · up/down `#1e5b2a`/`#a33430`.
- **Neutral ramp:** `#f6f7f9 → #eef1f5 → #e6eaef → #dde3ea → #6b7280 → #111827`; on-navy text `#fff / #cfd8e3 / #a9bfd1`.
- **Banned:** purple-gradient defaults; cream/parchment body; any new hex outside the token file.

## Spacing & grid

- Base unit 4px; committed steps **4 / 8 / 12 / 16 / 24 / 32**; 1040px content wrap.
- Density is a feature: tables run tight (10px 8px cells), prose caps at 65ch, whitespace spends itself on the signature displays, not between table rows.
- Cards only as data containers (tables, panels); never nested; full-bleed tables keep 14px first/last-column inset.

## Motion character

- **Personality: the score-bug.** 150ms, `cubic-bezier(0.22, 1, 0.36, 1)`, state-conveying only (hover, sort, tab, filter). No entrance choreography, no scroll-triggered reveals — content is already there.
- **The one sanctioned ambient motion — the broadcast ticker** (2026-07-23, owner): the recent-games row on the home/Stats page scrolls continuously right-to-left like an ESPN bottom-line. This is the *only* exception to "state-conveying only / ≤400ms"; it earns it by *being* the score-bug identity literally, not decorating it. Rules that keep it honest and usable: constant ~55px/s (duration derived from measured width, never a magic number); pauses on hover/focus-within so its cards stay clickable; a seamless two-copy loop (translate by exactly one copy's width, so it never gaps); edge-fade mask like a real lower-third; **no "LIVE" pip or real-time claim** (data is nightly); and under `prefers-reduced-motion: reduce` it collapses to a static, hand-scrollable strip. Standouts are never stripe-marked — loudest-first order and gold "hot" stat numbers surface them, ticker-uniform. Do not generalize this into other ambient motion anywhere else.
- **Signature moment:** "the group steps forward" — on the outlook field, interacting with a count plate flips it navy, recedes the other plates, and scales its detail panel out from its position with a ≤120ms card stagger; the whole sequence completes ≤400ms on the standard ease-out, closes in 150ms, and collapses to instant state under reduced motion. (The earlier glance-strip arc draw retains the same budget.)
- **Reduced motion:** every transition wrapped in `@media (prefers-reduced-motion: no-preference)`; the reduced experience is instant state, never a broken one.

## Information design

- **Encodings:** position/length for comparison (sliders, pool bar, the roster-math strip — 19+21=40 against a marked 34-man-limit tick, sourced to the NCAA); lines for trends (sparklines, narrative chart); the ramp as reinforcement, never the sole channel (numbers always present); **maps for geography** (built 2026-07-23 — the By league map, below); **the field's geometry used literally** — the Roster Outlook field (final form 2026-07-22): a pure park render whose only marks are five **fanned card stacks** at true positions — OF deep, **IF at second base**, **P on the mound** (owner reversed the earlier pitchers-off-field rule), **C at home plate**, **UTIL at the painted on-deck circle** in foul-side dirt. Each stack fans 2–3 real mini-portraits under a navy count badge, with a small navy **group-name pill beneath it** (full words desktop — Outfield/Infield/Pitchers/Catchers/Utility; OF/IF/P/C/UTIL abbreviations on the zoomed mobile park) so a first-time viewer reads the clusters without opening one; the label steps aside the instant a hand is opened (the panel is titled). Location plus label name the group; the fan is the interaction invitation and it spreads on hover/focus (the explicit "hover a group" cue was removed 2026-07-23 as redundant). The **park watermark shows on mobile too** (the 150% mobile zoom renders it smaller than desktop, so it never crowds). A stack expands into **a hand of gold-framed trading cards** held over the scrimmed park (portrait anchor, name, pos · class, chip, + badge): cards are uniform (112×176, 2px gold frame), rows cap at seven at near-full visibility with percentage-based offsets (a clamped viewport compresses the fan, never spills it), bigger groups deal into stacked rows, pitchers deal into Left-handed/Right-handed sub-hands under gold-ruled dividers, hand headers use full group names (Infield, Pitchers, Catchers, Outfield, Utility), and hovering/focusing a card slides it up out of the hand to read. Phones fold the hand into a sectioned list. Status dots were tried and retired — with panels carrying the detail they read as decoration, and the motif must encode. The field is a *positional* canvas, not an abstract axis: the percentile-arc experiment was rejected. Each field use must encode real data; motif-as-decoration is banned.
- **Disclosure:** 5s (glance strip + sorted table) → 1min (movers, filters, league context) → 5min (player deep-dives, draft money). Overview first, filter, details on demand.
- **IA (2026-07-23, owner):** the **Stats dashboard is the home page** — nav order **Stats (/) · Roster Outlook (/outlook) · Draft · By League** (owner reverted an interim Outlook-as-home experiment same day). Stats owns the `/players/*` deep-dives it links to (nav stays lit there; the player back-link and `/stats` both resolve home). Roster Outlook keeps its own route: on desktop the park is capped at 780px and the vertical chrome is trimmed so h1 + roster-math strip + section header + the full park share one ~900px screen; the 20-man pitchers hand deals compact (100×158 cards, tighter rows) to stay inside the capped frame, and the park watermark sits in the open grass between the OF and IF stacks.
- **Annotation layer (numeric, not verbal):** signature displays state their takeaway through marks, chips, and labeled numerals — a leader dot labeled "LEWIS · 96", a stat pill, a highlighted tile — never through generated prose. Labels, not sentences; the longest permitted annotation is a terse label + figure. The nightly refresh is visible (ticker, movers, "stats through" stamps).
- **Integrity (non-negotiable):** no fake data, ever — fixture stats never render; every reported figure links its source; "unverified" stays labeled; empty states say why they're empty.

## Titles & labels

Section titles are plain descriptive nouns: "Player statistics", "Game log", "2027 roster by position", "Last night". Never wordplay, slogans, or broadcast-catchy phrasing ("Every Jacket, Every League" was rejected as corny — confirmed 2026-07-21). The voice of a serious data site: the displays are expressive, the labels are not.

## Player imagery

One player mark, sized to surface (revised 2026-07-22):
- **Bust cards (4:5 rounded rectangles) are the universal player presentation** — score cards, the outlook field, staff panel, player heroes, statistics tables, roster lists. One shape = one identity language; size varies (~92px heroes → ~30px list rows). Circles are retired.
- **Percentile encoding by context:** micro-bars (length + ramp + numeral, `PercentileBar`) in comparison columns; pills (`PercentileChip`) only in tight contexts (hero tiles, inline-with-name chips).
- **Framing spec: top of hat = top of frame**, chin near the bottom. Values are face-detection derived per photo (OpenCV boxes → geometric viewport), regenerated by the crop tooling and verified against rendered contact sheets — never eyeballed.
- **Provenance speaks from the + badge, not from pills** (final 2026-07-22): the navy + at a portrait's bottom-left marks players new to GT, and the site's own navy tooltip (250ms intent, 150ms fade, shown on portrait hover and on row-link focus — never the OS's slow native title) carries the origin ("Incoming transfer from Jacksonville State", "Incoming freshman · Etowah HS" — `provenanceLabel`). School/FR pills are retired from every roster list; the only visible tag left is the amber drafted-unsigned limbo (a warning, not a footnote) and the draft page's GT-signee role mark. Dashed frames persist as arriving reinforcement in panels.
- **The + arrival mark lives on the portrait, not in the tag** (moved 2026-07-22, + at bottom-left per owner): a small navy corner badge on the bust of every player new to GT for next season; provenance tags carry only the word (school / FR / GT signee). Drafted-but-unsigned players never get the badge — their arrival is an open question.
- Full-body action shots render as gold monogram tiles; a blurred face zoom is worse than initials.

## Component voice

- **The draft page is the Class of 2026 card set** (rebuilt 2026-07-23): gold-framed cards in pick order with navy pick-corner tags, the bonus as each card's receipt-linked headline numeral, sign-date micro-stamps; the amber limbo card carries the countdown while the deadline pends and becomes the resolution card after (a signee who doesn't sign reads "Arrived at GT" — never an error state). Money is length-encoded twice, honestly: the pool band is one segment per signing (slot total as a labeled reference, never a denominator) and the signing-bonus ladder puts every figure on one linear dollar scale with navy slot ticks (no tick = no assigned slot). The receipts legend lives under the card set (the ledger table was retired as redundant 2026-07-23). The signing-bonus ladder's key is a **visual legend** (2026-07-23) — a gold-bar swatch and a navy-tick-on-track swatch that mirror the row marks, not a sentence describing them. Money is never ramp-colored — it is not performance.
- **The By league page is a map led by league** (rebuilt 2026-07-23, replacing a generic card grid the critique scored 18/28 as the least-authored surface). The signature is `LeagueMap` — a flat US map (muted `--line-soft` land, white borders, MLB-app register) with every GT summer team plotted at its true city. Map land and dots share one Albers projection baked at build time by `pipeline/geo/build_usmap.py` (public-domain geometry → `site/src/data/usmap.json`); re-run it when the team set changes. **League is the organizing dimension, not performance:** every league owns a **categorical color** (`--lg-*`, muted heritage tones) carried across its map nodes, its map label, and its accordion code-chip — the shared visual key. Nodes are sized subtly by GT count and are never ramp-colored (this surface answers "who's in which league, and where," not "who's hottest"). Desktop labels **spell out the full league name** (long ones wrap two lines); phones fall back to the acronym. Labels are hand-placed in open space and their positions are regression-checked at build-review so **no label ever crosses a dot** (desktop + mobile, 0px).
  The seven Cape Cod towns cluster too tightly for the national scale, so the CCBL gets a **zoomed inset that is a real Cape Cod mini-map** (2026-07-23): a high-res MA coastline (`ma-coastline.geojson`) is clipped to the Cape and reprojected so the hooked peninsula shows and each town sits on true land; a magnifier-lens ties it to the Cape's national position; each town label points outward from the cluster into open water. On phones the inset can't shrink legibly, so the Cape collapses to one aggregate `CCBL · 10` node.
  Beneath the map the leagues are an **accordion** (sorted by GT count): each league is a collapsed 48px header row (colored code chip · full name · count · teams · chevron) that opens **one at a time**, its teams-by-city grid animating open via `grid-template-rows: 0fr→1fr` (300ms ease-out, instant under reduced motion). **The map drives it:** clicking a **team node** opens its league, scrolls the content fully into view (not just the header), and **spotlights that team's block** with a gold outline; a label or the Cape inset opens the league to the top. Focusing/hovering a league also spotlights its nodes and dims the rest ("step forward"). Players within each team sort by **percentile, then name**. **All rows start collapsed** (owner 2026-07-23 — Cape-open-by-default was rejected); the map + collapsed rows is the compact resting state. Percentiles use the app's standard **`PercentileBar`** (length + ramp + numeral, matching the Stats leaderboard — the bare muted chip was retired here as a one-off), kept muted for the small summer sample under a plain caption ("Rating is percentile within each player's summer league — OPS for hitters and ERA for pitchers"; the earlier "summer snapshot / deeper red" wording and the cold/hot swatch were cut as noise). Honest limbo/awaiting states, ≥44px rows, a real official-site hit area. No cross-league "hottest" strip — leaderboards live on Stats; this tab separates, it doesn't rank.
- **Tables** are the reference layer: quiet, dense, sortable, sticky-identity on mobile, percentile chips as the only color.
- **Percentiles carry their window** (owner flag 2026-07-23): a bare colored percentile chip reads as an overall grade. The Stats leaderboard keeps solid, high-authority ramp chips (sortable, the whole point of that surface); the Roster Outlook's Returning list uses **muted, outlined** chips under a plain caption — *"Rating is percentile within each player's summer league — OPS for hitters and ERA for pitchers."* Same ramp, lower authority, so a small summer-league sample is never mistaken for a career rating.
- **Percentile math is a true midrank vs. the QUALIFIED pool** (reworked 2026-07-23 per owner). Each league's full leaderboard is scraped, so a rating is a real rank, not a model — `pipeline/percentiles.py`. The **pool leader reaches 100** (in-pool ranking uses the n−1 denominator; a tied leader still splits, correct math). **One qualifying bar, site-wide** (`compute.py`, scaled to each team's games so it tracks season length): a hitter needs **2.0 PA per team game**, a pitcher **0.5 IP per team game** — below MLB's 3.1/1.0 rate-title bars because summer play is sporadic. `qualified` rides on every stat block. **Non-qualifiers still get a percentile** (ranked against the qualified pool) but the player-page slider fill is **hatched** (Savant convention) with a terse "Unqualified sample size" note, and the Stats table drops them below its "Below qualifying threshold" line. So a 0.00 ERA in 1.2 IP reads ~98 **hatched**, not a clean 95. Requires a scrape to recompute (the pool isn't persisted); Galason's CPL line is the one manual entry the microservices feed can't reach, carried forward.
- **Honest summer-status groups** (renamed 2026-07-23): *On summer roster · no stats* (assigned, feed pending) · *No summer ball* · *Incoming freshmen*. "No summer ball" covers everyone without a summer club, whether one is still possible or none is expected — **we don't presume finality**: a graduated post-grad returner can still pick up summer ball (Gaudette lives here, not in a separate "not playing" bucket, since graduating doesn't end GT eligibility). Empty groups vanish; each figure keeps its per-player note as its receipt.
- **Chips** speak the ramp; **sliders/arcs** are the showpieces — gold dots with navy strokes on neutral tracks.
- **Provenance tags** (`ProvenanceTag`, added 2026-07-22) say where a player came from in ~one word, one grammar on every roster list: navy filled = incoming transfer (label is the origin school), gold outlined = incoming freshman/GT signee, amber = drafted-but-unsigned limbo. Incoming tags lead with a muted ▸ arrival mark; the limbo tag doesn't — arrival there is undecided. Returning players go untagged — the default population is silent. Categorical only: the ramp never encodes provenance, and provenance colors never encode performance.
- **Buttons/controls** are compact and navy-committed when active (filled), well-gray when idle; focus ring navy-on-light, gold-on-navy.
- **Heroes** are broadcast lower-thirds: condensed caps name, gold meta line, context right-aligned. **The player page IS a trading card** (final 2026-07-23): the profile container wears the card frame (4px gold, 16px radius, object shadow); the navy hero is the card front (portrait with the percentile grading-sticker on its corner + provenance badge); interior section titles are card-back navy bars; season stats ("Summer 2026 Stats") set as the classic card-back table — two counting lines + the slash line in condensed tabular numerals inside a gold-hairline panel (KPI tiles retired); the photo corner carries the position chip with the class written out (Sophomore, Redshirt Senior); the card-front bio strip lists HT/WT/B-T only where an official roster sources them (returners via ramblinwreck, freshmen via PG; transfer backfill pending); the frame carries the classic gold-edge/white-gutter/gold-pinstripe. Percentile rankings ("[League] Percentile Rankings") sit in their own tinted framed panel with the population note beneath; slider fills are the solid percentile color, not a ramp gradient. Rolling OPS / Rolling ERA is the Savant-idiom rolling chart (smoothed, dotless): dashed y-gridlines with condensed axis values (step scales to the range), the league average dashed and named LG AVG at the right edge, and the line stroked in the tested percentile ramp by distance from average — inverted for ERA, numbers always present via axis + headline. HT/WT sit above Summer 2026 Stats; the photo corner carries the position chip alone. It and the Game Log (weight-only emphasis) live outside the card as plain sections.
- Empty states teach ("stats land with the nightly refresh"), never apologize.

## Anti-patterns (this design must never)

- Show placeholder/sample stats as if real (hard interview rule).
- Put body text on gold, or white text on any untested ramp color.
- Use the field motif decoratively (turf textures, stitched-leather skeuomorphism, diamonds that encode nothing).
- Add a third type family, hero-metric gradient cards, side-stripe borders, or gradient text.
- Animate entrances/scroll reveals; exceed 400ms on any motion.
- Lead with text where a display can carry it (the site is "NOT text-heavy") — but every display keeps its numbers visible.
- Generate prose from data (verbal headlines, adjective performance lines). The site never *writes about* the numbers; it stages them. Confirmed correction 2026-07-21: even the hero speaks in headshot + numerals + labels.
- Introduce hexes outside the token file (the detector + tests are the enforcement).

## Taste log

- **Likes (confirmed 2026-07-21):** Savant idiom; MLB Gameday's field-as-visual; condensed athletic caps; navy-dominant heroes with white data surfaces; recurring field canvas; illustrative density; screenshot-worthy displays as the brand.
- **Dislikes (confirmed 2026-07-21):** FanGraphs' look ("presenting similar things" but disliked); corporate/minimalist/text-heavy registers; fake data in any form; the dossier's serif voice and the scoreboard's gold-drenched surfaces (specimen losers — B "too quiet/print," C "too costume"); **generated verbal headlines/adjective lines — the user vetoed prose-led displays mid-build: visuals and stats talk, words are labels**; **"star of the night" editorial crowning — show everyone's night as a scores panel, order by loudness, never crown**; **the percentile arc/"half wheel" — rejected; the ballpark visual means positions (roster on a field), not a bent axis.**
